# Customer Churn Prediction App

**Predict whether a telecom customer will churn** using machine learning and an interactive web app.

## Overview

End-to-end project:
- Data cleaning & EDA on Telco Customer Churn dataset
- Label encoding & handling imbalance with class_weight
- Random Forest Classifier (tuned with GridSearchCV)
- Custom decision threshold (0.45) for better recall
- Streamlit web app for interactive predictions

## Results (at threshold 0.45)

-## Model Performance (threshold = 0.45)

- **Churn Recall**: 57.6% → catches more than half of customers who actually churn  
- **Precision**: 26.7% → about 1 in 4 predicted churners actually leave  
- **Accuracy**: 44.9% (lower due to prioritizing recall over majority-class accuracy)  

**Why this is valuable**: In churn prediction, missing a customer who will leave is far more costly than contacting someone who stays.  
By lowering the decision threshold from 0.5 → 0.45, recall increased ~2.6× while keeping precision reasonable.
